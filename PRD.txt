Product Requirements Document (PRD)1. Product Vision & Purpose
The objective is to build an AI-powered B2B account research agent that automates the deep-dive investigation of enterprise companies for sales opportunities. The agent must execute a strict, 13-step sequential process to gather firmographics, organizational structure, technology stack usage, and outsourcing models, ultimately generating a structured 33-column Excel report.2. Target AudienceEnterprise Sales RepresentativesAccount ExecutivesB2B Market Researchers3. Core User FlowInput: The user enters a target company name into the interface.Processing: The system visually indicates the agent's progress as it moves through the 13 defined steps.Review & Export: The system presents a summary of the findings and provides a downloadable Excel file containing the 33 strictly formatted columns.4. Key Functional Requirements (The 13-Step Agent Engine)Initial Classification Engine (Step 1): The system must cross-reference the input against an existing Excel database of Pega Partners and Customers. If identified as a Partner, the agent must immediately halt execution and output the limited data.Revenue & Industry Analysis (Steps 2-3): The agent must scrape the company's official website and, if necessary, query an LLM to determine if the core business is "Software" or "Non-Software". It must also extract the industry, HQ location, and annual USD revenue.Corporate Hierarchy & Localization Mapping (Steps 4-5): The system must use Google Search to identify parent companies and any India-based subsidiaries. If an India subsidiary exists, the agent must pivot its focus to collect data specifically from this entity. It must also verify the existence and location of Global Capability Centres (GCCs) in India.Social Graph Verification & Headcount (Steps 6-8): The agent must locate the correct LinkedIn pages for the parent and subsidiary entities. It needs to verify the industry and HQ against earlier findings and extract total headcount. Furthermore, it must programmatically filter the employee base to count "Engineering", "IT", "QA", and "SDET" roles, both globally and specifically for India.Tech Stack & Pega Validation (Steps 9-10): The system must perform targeted keyword searches within the employee base for terms like "Pega", "PRPC", "ServiceNow", and "Salesforce". It must apply a confidence threshold, requiring 5-10 profiles with the technology in their current role to confirm usage.Outsourcing & Final Categorization (Steps 11-13): The agent must detect relationships with major IT service firms (e.g., TCS, Infosys, Accenture) to determine the software development model. Based on all gathered signals, the system executes logic to classify the enterprise as E1 (Fully Outsourced), E1.1 (Transitioning), E2 (Non-Software), or E3 (Software-First). All discrepancies must be logged in a dedicated "Research Notes" field.Skills and Technology Stack RequiredTo build this, you will need a mix of frontend web development and heavy backend data engineering. Python is the industry standard for AI agent orchestration, and JavaScript will be perfect for building the user interface.1. AI & LLM Orchestration (Python)You need a framework to control the flow of the 13 steps, allowing the LLM to decide when to search, when to scrape, and when to finalize an answer.Skills: Prompt engineering, function calling, state management.Tools: LangChain or LlamaIndex (Python frameworks for building AI agents). OpenAI API or Anthropic API for the core reasoning engine.2. Web Scraping & Search IntegrationThe agent relies heavily on live web data and Google searches to find subsidiaries and tech evidence.Skills: DOM parsing, handling HTTP requests, API integration.Tools: BeautifulSoup or Scrapy (Python) for standard website text extraction. Google Custom Search JSON API (or SerpApi) to execute the required Google searches programmatically.3. LinkedIn Data Extraction (The Hardest Part)Steps 6 through 10 require highly specific data from LinkedIn (employee counts, role filters, keyword searches). Official LinkedIn APIs do not allow this level of scraping for third-party apps.Skills: Bypassing anti-bot protections, utilizing third-party data enrichment APIs, or browser automation.Tools: Proxycurl or Nubela APIs (third-party services that provide structured LinkedIn profile and company data). Alternatively, browser automation tools like Playwright or Selenium, though this approach is fragile and prone to rate limits or account bans.4. Data Structuring & ExportThe application must strictly format the output into 33 specific columns.Skills: Data wrangling, tabular data manipulation.Tools: Pandas (Python library) to structure the scraped data into DataFrames and easily export it using to_excel().5. Web Development (Frontend & API)You need a web interface where users can input the company name and download the final Excel sheet.Skills: RESTful API creation, asynchronous JavaScript.Tools: FastAPI or Flask (Python) to serve as the backend API communicating with your agent. JavaScript, HTML, and CSS (or a framework like React) for the frontend dashboard.
